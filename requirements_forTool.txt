Phase 2 — Backend data pipeline
Tell Claude Code to build the data layer in this order:
Step 1 — CSV ingestion script
Prompt Claude Code:
"Write a Node.js script in scripts/ingestCSV.js that:
- Reads all CSV files from a /data folder
- Parses them with papaparse
- Maps each row to a release record with fields:
  projectName, releaseVersion, testPassRate, automationCoverage,
  criticalBugsOpen, totalBugs, escapedDefects, mttr,
  issues[] (each with id, title, severity, rca, capa, owner, dueDate, team)
  teamLearnings[] (team, learning, owner, dueDate)
- Saves the parsed data to a local SQLite DB (or JSON store for v1)"
Step 2 — Score calculation engine
Prompt Claude Code:
"Write scorecard.js that takes a release record and returns a
score 0–100 based on weighted inputs:
  testPassRate        → 25%
  automationCoverage  → 20%
  criticalBugsOpen    → 20% (inverted — 0 bugs = full score)
  escapedDefects      → 20% (inverted)
  slaAdherence        → 15%
Return score, breakdown object, and a go/nogo/conditional recommendation."
Step 3 — Express API
Prompt Claude Code:
"Write server/index.js with these routes:
  GET /api/projects          → list all projects + latest release score
  GET /api/projects/:id      → full scorecard for one project
  GET /api/projects/:id/history → score trend across releases
  POST /api/projects/:id/capa → submit/update a CAPA entry
  GET /api/export/:id/pdf    → trigger PDF export of scorecard"

Phase 4 — JIRA integration
Step 7 — Live bug sync
Prompt Claude Code:
"Write server/jira.js that:
- Authenticates with JIRA using API token from .env
- Fetches all bugs for a given project key and fix version:
    open criticals, total filed, resolved count, severity breakdown
- Maps JIRA issue fields to the release record schema
- Is called on a 5-minute polling interval and on-demand via
  POST /api/projects/:id/sync
- Caches results in Redis (or in-memory Map for v1) with 5min TTL"
